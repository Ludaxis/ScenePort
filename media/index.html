    Media assets¶

    This folder holds the images and the hero clip referenced by the root README.md, the docs site (docs/index.md), and the recipe pages. The files are intentionally not committed as placeholders; drop the real assets here and they render automatically.

    Files and where they appear¶

    File Used by How to produce it
    hero.gif README.md hero slot, docs/index.md Record per HERO_STORYBOARD.md, export as GIF.
    hero.mp4 README.md MP4 fallback (commented) Same recording, export as MP4.
    create-ui-from-screenshot.png "What you can do" + recipe Run a UI-from-screenshot pass, then scripts/capture-demo.mjs game.
    self-heal.png "What you can do" + recipe Capture the Game view after a self-heal loop clears the console.
    visual-regression.png "What you can do" + recipe Use the diff image from unity_compare_golden_frame, or a Game-view capture.
    explain-scene.png "What you can do" + recipe Capture the Scene/Game view of an explained scene.
    game-view.png generated by capture harness scripts/capture-demo.mjs game
    scene-view.png generated by capture harness scripts/capture-demo.mjs scene

    Generating screenshots automatically¶

    With a Unity Editor open and the ScenePort bridge running:

    cd plugins/sceneport/server
SCENEPORT_PROJECT_PATH=/path/to/UnityProject \
  node ../../../scripts/capture-demo.mjs        # writes docs/media/game-view.png + scene-view.png

    See the header of scripts/capture-demo.mjs for options. Rename or copy the generated PNGs to the feature-specific names above as needed.

    Guidelines¶

    • Keep PNGs reasonably sized (the capture harness defaults to maxEdge: 1024).
    • Prefer a 16:9 Game view for the hero clip and feature shots.
    • The README uses <img>/Markdown image tags, so missing files degrade to alt text rather than breaking the page.
